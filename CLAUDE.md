# Bowling Kata (AI + TDD 실습)

## 프로젝트 개요

Robert C. Martin의 Bowling Game Kata를 AI 기반 TDD로 연습하는 프로젝트다.
언어/테스트 프레임워크: **Python + pytest**.

## 요구사항

`Game` 클래스를 작성한다.

- `void roll(int pins)`: 투구마다 호출된다. 인자값은 쓰러뜨린 핀의 수다.
- `int score()`: 게임의 총점을 계산하여 반환한다.

범위에서 제외되는 것 (kata 단순화를 위해):

- 투구가 유효한지 검증하지 않는다.
- 투구 횟수나 프레임 수가 정확한지 검증하지 않는다.
- 중간 프레임의 점수를 제공하지 않는다.

## 점수 규칙 요약

- 게임은 10프레임, 각 프레임 최대 2회 투구(10번 프레임 제외).
- **Spare**: 한 프레임의 두 투구 합이 10. 보너스 = 다음 1회 투구 핀 수.
- **Strike**: 첫 투구에서 10. 해당 프레임은 1회 투구로 종료. 보너스 = 다음 2회 투구 핀 수 합.
- **10번 프레임**: Spare/Strike 시 보너스용 추가 투구를 포함해 최대 3회까지 투구 가능. 단, 그 보너스 투구 자체가 다시 Strike/Spare여도 추가 기회는 없다.

## 개발 워크플로

이 프로젝트에서 코드를 작성할 때는 반드시 `skills/tdd-skill/SKILL.md`의 **테스트 주도 개발(TDD)** 규칙을 따른다.

핵심 원칙 (철의 법칙): **실패하는 테스트가 먼저 존재하지 않는 한, 프로덕션 코드를 작성하지 않는다.**

사이클:

1. RED — 실패하는 테스트 하나를 작성한다.
2. RED 검증 — `pytest`를 실행해 예상한 이유로 실패하는지 직접 확인한다.
3. GREEN — 테스트를 통과시키는 최소한의 코드만 작성한다.
4. GREEN 검증 — `pytest`를 실행해 전체 테스트가 통과하는지 확인한다.
5. REFACTOR — 그린 상태를 유지하면서 중복 제거, 이름 개선 등 정리한다.
6. 다음 요구사항으로 반복한다.

테스트 실행:

```bash
pytest
pytest -v
pytest -x
```
